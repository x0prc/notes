## Executive Process
- `_EPROCESS` in kernel memory.
- Information about the process, pointers to other relevant structures are the contents of this.
- Each process has a `_PEB` process environment block in memory.
- `PsActiveProcessHead` , `PsIdleProcess`
- `_KPRCB` contains a pointer to the current thread's `_KTHREAD`.
- `_EPROCESS` contains a pointer to the linked list.

## Kernel Bugs Exploitation
- Allocate virtual memory at any address in the bottom 4k
	- Use `NtAllocateVirtualMemory()`
	- Standard `VirtualAlloc()` fails
- Control the data referenced from userland (fake data).
- Trigger vulnerable code.
- [Kernel Pointer Leak](https://www.ragestorm.net/blogs/?p=486) 
- Exploiting UAF / type confusion, integer bugs and overflows, race conditions, memory information disclosure, logical only bugs.
- [Chrome Zero Day Exploit](https://securelist.com/puzzlemaker-chrome-zero-day-exploit-chain/102771/)
## Mitigation
- No eXecute bit added to Intel/AMD CPUs.
- **KASLR** : Kernel Address Space Layout Randomization
	- Useful for remote kernel vulnerabilities.
- **SMEP** : Supervisor Mode Execution Protection.
- Object `TypeIndex` field encoding.
- **Kernel Safe Unlinking** used for removing an element from a `_LIST_ENTRY` triggers a mirror write.
- **Kernel Virtual Address Shadow** used as swap to different page tables.
